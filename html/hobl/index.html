Summary of Key Concepts

Service A resource or functionality made available to users to perform specific tasks or fulfill particular needs.

Burn Rate The rate at which the error budget is consumed.

Key Concepts in Detail

Service Level Objective (SLO)

SLOs are measurable targets expressed in percentages, helping ensure expected performance and reliability for users. For example, an SLO might specify 99.5% uptime over a year or 97% of web pages loading within 3 seconds over a month.

Components of SLOs:

  • Service
  • Level
  • Objective

Service Level Indicator (SLI)

SLIs are metrics representing the performance of a service. For example, availability, latency, or error rate. SLIs are used to calculate SLOs:

Good Events / Total Events * 100

Key elements to define SLIs:

  • Type of service observed
  • SLI metric
  • Time window

Common SLI Metrics

Type of Service SLI Metric Description SLO Example
Request-driven Availability The percentage of time the service is available without failures. 99.5% uptime over the year.
Request-driven Latency The time taken by a service to respond to a request. 95% of web pages should load within 2 seconds over a 30-day period.

SLO Time Windows

Time windows define the period for which an error budget is calculated. Common types:

  • Rolling window: Continuously moving time window.
  • Fixed window: Has a defined start and end date.

SLO Challenges

Common challenges include:

  • Overly complex SLOs
  • Choosing the right metrics
  • Ensuring accurate measurement

Best Practices

Recommendations for effective SLOs:

  • Select metrics that matter to users.
  • Start simple.
  • Set realistic targets.
  • Review and improve regularly.
